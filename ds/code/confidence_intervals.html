Intervalos de Confiança¶

Média da amostra e média populacional¶

A mdia das amostras e a mdia populacional so diferentes. Geralmente, queremos saber sobre uma mdia de populao, mas podemos calcular apenas uma mdia de amostra. Usamos intervalos de confiana na tentativa de determinar com quanta preciso nossa mdia estimada da amostra qaproxima a mdia da populao.

Intervalo de confiança¶

Se fosse requisitado, por exemplo, que fosse feita uma estimativa da altura mdia de uma mulher nos Brasil, poderiamos fazer isso tomando a altura de 10 mulheres e estimando que a mdia dessa amostra era prxima da populao. Vamos tentar isso.

Isso ainda no faz tanto por ns, para realmente ter uma noo de como a mdia da amostra se relaciona com a mdia da populao precisamos calcular o erro padro. O erro padro uma medida da varincia da mdia da amostra.

IMPORTANTE¶

A computao de um erro padro envolve assumir que a maneira como voc amostra no adiciona vis e que os dados so normais e independentes. Se essas condies forem violadas, seu erro padro estar errado. Existem maneiras de testar e corrigir isso.

A frmula para erro padro .

$$ SE = \frac {\sigma} {\sqrt {n}} $$

Onde $ \sigma $ o desvio padro da amostra e $ n $ o nmero de amostras.

In [4]:
SE = np.std(heights) / np.sqrt(sample_size)

Existe uma funo na biblioteca de estatsticas de Scipy para calcular o erro padro. Observe que esta funo, por padro, contm uma correo de graus de liberdade que muitas vezes no necessria (para amostras suficientemente grandes, efetivamente irrelevante). Voc pode omitir a correo configurando o parmetro ddof para 0.

Supondo que nossos dados sejam normalmente distribudos, podemos usar o erro padro para calcular nosso intervalo de confiana. Para fazer isso, primeiro estabelecemos o nvel de confiana desejado, digamos 95%, ento determinamos quantos desvios padro contm 95% da massa. Acontece que os 95% da massa situa-se entre -1,96 e 1,96 em uma distribuio normal normal. Quando as amostras so grandes o suficiente (geralmente> 30 tomado como um limiar), o teorema do limite central aplica-se e a normalidade pode ser assumida com segurana; Se o tamanho da amostra for menor, uma abordagem mais segura usar uma distribuio $ t $ com graus de liberdade adequadamente especificados. A maneira real de calcular os valores usando uma funo de distribuio cumulativa (CDF). Se voc no est familiarizado com CDFs, CDFs inversos e seus PDFs complementares, voc pode ler sobre eles [aqui] (https://en.wikipedia.org/wiki/Probability_density_function) e [aqui] (https: // en. wikipedia.org/wiki/Cumulative_distribution_function). Olhe [aqui] (https://en.wikipedia.org/wiki/Student%27s_t-distribution) para obter informaes sobre a distribuio $ t $. Podemos verificar o nmero de 95% usando uma das funes do Python.

Erros comuns¶

Agora, ao invs de relatar a mdia da amostra, sem qualquer medida da probabilidade de esta estar correta, podemos calcular um intervalo e ser muito mais confiantes de que a mdia da populao se encontra nesse intervalo. Para fazer isso, tomamos a mdia da amostra $ \mu $ e informamos $ \left (\mu-1.96 SE, \mu + 1.96SE \right) $.

Isso funciona porque, assumindo a normalidade, esse intervalo conter a mdia da populao 95% do tempo.

Vale notar que o valor real da estimativa e os intervalos de confiana so fixos. incorreto dizer que "A mdia nacional da altura da populao do sexo feminino esta entre 1,5m e 1,7, com 95% de probabilidade.", mas infelizmente este um erro comum. Ao invs disso, o "95%" se refere ao fato de que dentro de muitos calculos do intervalo de confiana de 95%, o valor real estar dentro do intervalo em 95% dos casos (assumindo valibrao correta do intervalo de confiana, o que discutiremos a seguir). Mas de fato, para uma nica amostra e um nico intervalo de confiana calculado desta, no temos maneiras de estimar a probabilidade do intervalo conter a mdia da populao. A visualizao abaixo demonstra este fato.

No bloco de cdigo abaixo, h duas coisas a serem observadas. Primeiro, embora o tamanho da amostra seja suficientemente grande para assumir a normalidade, estamos usando uma distribuio $ t $, apenas para demonstrar como ela usada. Em segundo lugar, os valores de $ t $ necessrios (anlogos aos $ \ pm1.96 $ usados acima) esto sendo calculados a partir da funo de densidade cumulativa invertida, o ppf em scipy.stats. A distribuio $ t $ requer os graus de liberdade de parmetro extra (d.o.f), que o tamanho da amostra menos um.

np.random.seed(8309)
n = 100 # number of samples to take
samples = [np.random.normal(loc=0, scale=1, size=100) for _ in range(n)]

fig, ax = plt.subplots(figsize=(12, 7))
for i in np.arange(1, n, 1):
    sample_mean = np.mean(samples[i])  # calculate sample mean
    se = stats.sem(samples[i])  # calculate sample standard error
    h = se*stats.t.ppf((1+0.95)/2, len(samples[i])-1) # calculate t; 2nd param is d.o.f. 
    sample_ci = [sample_mean - h, sample_mean + h]
    if ((sample_ci[0] <= 0) and (0 <= sample_ci[1])):
        plt.plot((sample_ci[0], sample_ci[1]), (i, i), color='blue', linewidth=1);
        plt.plot(np.mean(samples[i]), i, 'bo');
    else:
        plt.plot((sample_ci[0], sample_ci[1]), (i, i), color='red', linewidth=1);
        plt.plot(np.mean(samples[i]), i, 'ro');
plt.axvline(x=0, ymin=0, ymax=1, linestyle='--', label = 'Mdia da populao');

O que isto significa?¶

Os intervalos de confiana nos permitem definir nossa confiana desejada e, em seguida, informar um intervalo que provavelmente conter a mdia da populao. Quanto maior nossa confiana desejada, maior o alcance relatado. Em geral, nunca se pode relatar um nico valor pontual, porque a probabilidade de que qualquer ponto seja a mdia real da populao incrivelmente pequena. Vamos ver como nossos intervalos diminuem quando mudamos o tamanho da amostra.

Má Calibração e Violação de Suposições¶

O clculo dodesvio padro, erro padro e intervalo de confiana depende de certas suposies. Se essas suposies forem violadas, o intervalo de confiana de 95% no conter necessariamente o parmetro da populao 95% do tempo. Dizemos que, neste caso, o intervalo de confiana mau calibrado. Exemplificamos a seguir:

Exemplo: Dados Autocorrelacionados¶

Se o processo de gerao de dados for autocorrelacionado, as estimativas do desvio padro sero incorretas. Isso ocorre porque os processos autocorrelacionados tendem a produzir valores mais extremos do que os processos normalmente distribudos. Isso ocorre porque os novos valores dependem de valores anteriores, entao as sries que j esto longe da mdia provavelmente ficaro longe da mdia. Para verificar isso, geraremos alguns dados autocorrelacionados de acordo com o processo a seguir.

$$X_t = \theta X_{t-1} + \epsilon$$ $$\epsilon \sim \mathcal{N}(0,1)$$

In [13]:
def generate_autocorrelated_data(theta, mu, sigma, N):
    # Initialize the array
    X = np.zeros((N, 1))
    
    for t in range(1, N):
        # X_t = theta * X_{t-1} + epsilon
        X[t] = theta * X[t-1] + np.random.normal(mu, sigma)
    return X

X = generate_autocorrelated_data(0.5, 0, 1, 100)

Agora que sabemos a mdia da populao, podemos verificar a calibrao dos intervalos de confiana. Primeiro, vamos escrever duas funes auxiliares que calculam um intervalo para alguns dados de entrada e verificar se o intervalo contm a mdia real, 0.

In [16]:
def compute_unadjusted_interval(X):
    T = len(X)
    # Compute mu and sigma MLE
    mu = np.mean(X)
    sigma = np.std(X)
    SE = sigma / np.sqrt(T)
    # Compute the bounds
    return stats.norm.interval(0.95, loc=mu, scale=SE)

# We'll make a function that returns true when the computed bounds contain 0
def check_unadjusted_coverage(X):
    l, u = compute_unadjusted_interval(X)
    # Check to make sure l <= 0 <= u
    if l <= 0 and u >= 0:
        return True
    else:
        return False

Agora vamos executar muitas tentativas, em cada uma vamos amostrar alguns dados, calcular um intervalo de confiana e, em seguida, verificar se o intervalo de confiana contm a mdia da populao. Manteremos uma contagem contnua e devemos esperar que 95% das tentativas sejam bem-sucedidas se os intervalos forem calibrados corretamente.
