Análise de Resíduos¶

No processo de anlise de resultados de algoritmos de aprendizado de mquina, uma grande fonte de informaes contida nos resduos do modelo. Resduos so as diferenas entre os resultados esperados e os resultados inferidos do modelo. Uma anlise minuciosa aqui pode revelar processos de overfiting ou underfiting e quantificar a parcela da estrutura presente nos dados que foi capturada pelo modelo.

Para realizar a anlise dos resduos, fazemos uso de ferramentas estatsticas bem conhecidas.

Regressão Linear¶

A regresso linear uma das tcnicas de modelagem mais fundamentais. A utilizamo para estimar uma relao linear entre um conjunto de variveis independentes $ X_i $ e uma varivel de resultado dependente $ y $. Nosso modelo assume a forma de: $$ y_i = \beta_{0} 1 + \beta_{i, 1} x_{i, 1} + \dots + \beta_{i, p} x_{i, p} + \epsilon_i = x_i'\beta + \epsilon_i $$

Para $i \in \{1, \dots, n \}$, onde $ n $ o nmero de observaes. Escrevemos em forma vetorial como:

$$ y = X\beta + \epsilon $$

Onde $ y $ um vector $ n \times 1 $, $ X $ uma matriz $ n \times p $, $ \beta $ o vetor de coeficientes $ p \times 1 $ e $ \epsilon $ um termo de erro amostrado de uma distribuio normal. Normalmente chamamos um modelo com $ p = 1 $ de regresso linear simples e um modelo com $ p> 1 $ de regresso linear mltipla.

Sempre que construmos um modelo, haver lacunas entre o que um modelo prediz e o que observado na amostra. As diferenas entre esses valores so conhecidas como os resduos do modelo e podem ser usadas para verificar algumas das premissas bsicas que se inserem no modelo. Os principais pressupostos a serem verificados so:

  • Ajuste linear: A relao subjacente deve ser linear
  • Homoscedasticidade: Os dados no devem ter tendncia na varincia
  • Independente e Distribudo de forma idntica: Os resduos da regresso devem ser independentes e distribudos de forma idntica (i.i.d.) e no mostrar sinais de auto-correlao

Podemos usar os resduos para ajudar a diagnosticar se o relacionamento que estimamos real ou esprio.

O erro estatstico uma mtrica similar associada anlise de regresso com uma diferena importante: enquanto os resduos quantificam a diferena entre as previses do modelo de regresso e a amostra observada, o erro estatstico a diferena entre um modelo de regresso e o valor esperado no observvel. Usamos resduos em uma tentativa de estimar esse erro.

Regressão Linear Simples¶

Para servir como modelo exemplo para nossa anlise, vamos definir uma regresso linear simples.

In [2]:
def linreg(X,Y):
    # Running the linear regression
    X = sm.add_constant(X)
    model = sm.OLS(Y, X).fit()
    B0 = model.params[0]
    B1 = model.params[1]
    X = X[:, 1]

    # Return summary of the regression and plot results
    X2 = np.linspace(X.min(), X.max(), 100)
    Y_hat = X2 * B1 + B0

    plt.scatter(X, Y, alpha=1) # Plot the raw data
    plt.plot(X2, Y_hat, 'r', alpha=1);  # Add the regression line, colored in red
    plt.xlabel('X Value')
    plt.ylabel('Y Value')
    plt.grid()
    return model, B0, B1

Vamos definir como exemplo uma relao entre $X$ e $Y$ que pode ser modelado com uma regresso linear. Definimos a relao e construimos um modelo em cima desta, plotando a linha de melhor ajuste dados os parametros da regresso.

Residuais¶

A definio de um residual a diferena entre o que observado na amostra e o que previsto pela regresso. Para qualquer residual $ r_i $, expressamos como:

$$ r_i = Y_i - \hat {Y_i} $$

Onde $ Y_i $ o valor observado de $ Y $ e $ \hat {Y} _i $ o valor previsto. Traamos essas diferenas no seguinte grfico:

Adequação de um Modelo Linear¶

Ao procurar padres em residuais, podemos determinar se o modelo linear apropriado ou no. Uma regresso linear simples no seria apropriada para uma relao subjacente da forma:

$$ Y = \beta_0 + \beta_1 X ^ 2 $$

Como uma funo linear no seria capaz de explicar completamente a relao entre $ X $ e $ Y $.

Se o relacionamento no um bom ajuste para um modelo linear, o grfico residual mostrar um padro distinto. Em geral, um grfico residual de uma regresso linear em uma relao no-linear mostrar vis e ser assimtrico em relao ao exiso residual = 0 enquanto um grfico residual de uma regresso linear em uma relao linear ser geralmente simtrico sobre o eixo residual = 0.

Como exemplo, consideremos uma nova relao entre as variveis $ X $ e $ Y $ que incorpora um termo quadrtico.

Heteroscedasticidade¶

Um dos principais pressupostos por trs de uma regresso linear que os dados subjacentes tm uma varincia constante. Se houver parte dos dados com varincia diferente do restante, os dados no so apropriados para uma regresso linear. Heteroscedasticidade um termo que se refere a dados com varincia no-constante, em oposio homocedasticidade, quando os dados apresentam variao constante.

A heteroscedasticidade significativa invalida os resultados da regresso linear ao adicionas vis ao erro padro do modelo. Como resultado, no podemos confiar nos resultados de testes de significncia e intervalos de confiana gerados a partir do modelo e seus parmetros.

Para evitar essas conseqncias, importante usar os residuais para verificar a heteroscedasticidade e ajustar o modelo, se necessrio.

Como um exemplo de deteco e correo da heterocedasticidade, consideremos ainda outra relao entre $ X $ e $ Y $:

Métodos estatísticos para detecção de heterocedasticidade¶

Geralmente, queremos dar suporte a observaes qualitativas sobre um grfico residual com um mtodo quantitativo. O plot residual nos levou a acreditar que os dados podem ser heterocedasticos. Vamos confirmar esse resultado com um teste estatstico.

Uma maneira comum de testar a presena de heteroscedasticidade o teste de hiptese Breusch-Pagan. bom combinar a anlise qualitativa de um grfico residual com a anlise quantitativa de pelo menos uma prova de hiptese. Ns tambm podemos adicionar o teste White, mas, por enquanto, usaremos apenas Breusch-Pagan para testar a correlao acima. Existe uma funo no pacote statsmodels chamado het_breushpagan que simplifica a computao:

Transformação Box-Cox¶

Finalmente, examinamos a transformao de Box-Cox. A transformao Box-Cox um mtodo poderoso que funcionar em vrios tipos de relaes heterocedasticas. O processo funciona testando todos os valores de $ \lambda $ dentro do intervalo $ [- 5, 5] $ para encontrar o valor que torna a sada da seguinte equao mais prxima para ser normalmente distribuda: $$ Y ^ {(\lambda)} = \begin {cases} \frac {Y ^ {\lambda} -1} {\lambda} &: \lambda \neq 0 \\ \log {Y} &: \lambda = 0 \end {cases} $$

O "melhor" $ \lambda $ ser usado para transformar a srie ao longo da funo acima. Em vez de ter que fazer tudo isso manualmente, podemos simplesmente usar a funo scipy boxcox. Utilizaremos estafuno para ajustar $ Y $ e remover a heteroscedasticidade.

Residuais e Autocorrelação¶

Outro pressuposto por trs de regresses lineares que os resduos no so auto-correlacionados. Uma srie auto-correlacionada quando esta est correlacionada com uma verso atrasada de si mesma. Um exemplo de uma srie potencialmente auto-correlacionada a maxima das temperaturas dirias. A temperatura de hoje nos d informaes sobre a temperatura de amanh com confiana razovel (as amostras de temperaturas tendem a serem proximas umas das outras, sem descontinuidades abruptas). Uma srie de rolagem de dados justos, no entanto, no seria autocorrelacionada, pois uma amostra no lhe d informaes sobre o prxima. Cada amostra independente da anterior.

Ns representamos uma dependncia de tempo em valores anteriores, da forma:

$$ Y_i = Y_{i-1} + \epsilon $$

Se os resduos de um modelo forem autocorrelacionados, voc poder fazer previses sobre os resduos adjacentes. No caso de $ Y $, sabemos que os dados sero autocorrelados porque podemos fazer previses baseadas em resduos adjacentes prximos uns dos outros.

Métodos estatísticos para detecção de autocorrelação¶

Tal como acontece com todas as propriedades estatsticas, exigimos um teste estatstico para finalmente decidir se h autocorrelao nos nossos resduos ou no. Para este fim, usamos um teste Ljung-Box.

Um teste de Ljung-Box usado para detectar autocorrelao em uma srie temporal. Este teste examina a autocorrelao em todos os intervalos de atraso abaixo de um mximo especificado e retorna sequencias contendo as sadas para cada intervalo de atraso testado.

Vamos usar a funo acorr_ljungbox do modulo statsmodels para testar a autocorrelao nos resduos do nosso modelo. Usamos um intervalo de atraso mximo de $ 10 $ e veremos se algum dos atrasos possui autocorrelao significativa:
